# Linear 코멘트 템플릿

맥락에 맞게 적응 가능한 가이드라인. 모든 코멘트는 한국어로 작성한다.

## Decision (결정)

```markdown
## 결정: {제목}

**결정**: {무엇을 결정했는지}

**배경**: {왜 이 결정이 필요했는지}

**검토한 대안**:
- {대안 1}: {선택하지 않은 이유}
- {대안 2}: {선택하지 않은 이유}

**근거**: {최종 선택 이유}
```

## Note (발견/메모)

```markdown
## 메모: {제목}

{발견 내용이나 참고사항}

**맥락**: {어떤 작업 중 발견했는지}
**영향**: {구현에 미치는 영향}
```

## Deviation (구현 차이)

```markdown
## 구현 차이: {제목}

**원래 계획**: {이슈 설명에서 의도한 내용}
**실제 구현**: {실제로 구현한 내용}
**변경 이유**: {왜 다르게 구현했는지}
```

## Session Wrap-up (세션 마무리)

```markdown
## 세션 요약 ({날짜})

### 작업 내용
- {작업 1}
- {작업 2}

### 결정사항
- {결정 1}
- {결정 2}

### 다음 단계
- [ ] {남은 작업 1}
- [ ] {남은 작업 2}
```
