# Review Postmortem

리뷰 코멘트를 시스템 개선으로 변환한다.

## 핵심 원칙

- **코더의 실수는 증상이지 원인이 아니다.** 실수가 가능했던 구조적 원인(빠진 문서, 부족한 예시, 스킬 누락)을 찾는다.
- **모든 리뷰 코멘트는 시스템 개선 기회다.** 같은 유형의 코멘트가 미래에 다시 필요하지 않도록 만든다.
- **AGENTS.md가 single source of truth다.** 컨벤션은 해당 폴더의 AGENTS.md에, 스킬은 그것을 참조하는 구조로 만든다.

## 실행 절차

### 1단계: 리뷰 코멘트 수집

사용자에게 리뷰 코멘트의 내용을 확인한다. PR URL이 주어지면 `gh` CLI로 코멘트를 가져온다.

```
gh api repos/{owner}/{repo}/pulls/{number}/comments
gh api repos/{owner}/{repo}/pulls/{number}/reviews
```

### 2단계: Five Whys 분석

각 리뷰 코멘트에 대해 근본 원인을 추적한다.

**분석 프레임워크:**

1. **무엇이 잘못되었나?** — 리뷰 코멘트의 내용
2. **왜 코더가 이렇게 작성했나?** — 참고한 예시나 문서가 있었는지
3. **왜 그 예시/문서가 잘못된 패턴을 보여주고 있었나?** — 문서가 업데이트되지 않았거나, 아예 없었거나
4. **어디에 올바른 패턴이 기록되어야 하나?** — AGENTS.md, 스킬, 메모리 중 어디
5. **이 변경으로 미래의 같은 실수를 막을 수 있는가?** — 검증 가능한지

### 3단계: 영향 범위 파악

리뷰 코멘트가 영향을 미치는 범위를 파악한다.

| 범위 | 저장 위치 | 예시 |
|------|----------|------|
| 특정 폴더/패키지 | 해당 폴더의 `AGENTS.md` | "react 패키지에서 splitVariantProps 사용 필수" |
| 프로젝트 전체 | 루트 `AGENTS.md` 또는 `TECH.md` | "모든 패키지에서 bun 사용" |
| 컴포넌트 생성 워크플로우 | 관련 스킬의 details/ | "create-component 스킬에서 AGENTS.md 참조" |
| 세션 간 지속 | 피드백 메모리 | "향후 대화에서도 반영되어야 하는 규칙" |

### 4단계: 시스템 개선 실행

분석 결과에 따라 아래 작업을 수행한다.

**AGENTS.md 업데이트:**
- 해당 폴더의 AGENTS.md에 컨벤션 규칙 추가
- 프로젝트의 기존 AGENTS.md 스타일을 따름 (코드 블록 유무, 산문체 등)
- 안티패턴이 있다면 "절대 금지" 섹션에 명시

**스킬 업데이트:**
- 관련 스킬이 해당 폴더의 AGENTS.md를 참조하도록 지시 추가
- 스킬의 details/ 파일에 구체적 규칙 반영
- 스킬이 AGENTS.md와 중복되지 않도록 주의 — 스킬은 AGENTS.md를 참조하는 구조

**피드백 메모리 저장:**
- 세션 간 지속되어야 하는 규칙은 memory에 feedback 타입으로 저장
- `**Why:**`와 `**How to apply:**` 라인 포함

### 5단계: 검증

- 변경된 AGENTS.md가 해당 폴더의 기존 스타일과 일관적인지 확인
- 스킬 변경이 있다면 skill-creator eval 사용을 제안
- 변경사항을 커밋하고 PR에 반영

## 출력 형식

포스트모템 완료 시 다음 형식으로 요약한다:

```
## 리뷰 포스트모템 결과

### 리뷰 코멘트
[원본 코멘트 내용]

### 근본 원인
[Five Whys 분석 결과]

### 시스템 개선
- [ ] AGENTS.md: [변경 내용]
- [ ] 스킬: [변경 내용]
- [ ] 메모리: [저장 내용]

### 검증
[이 변경으로 미래의 같은 실수를 어떻게 방지하는지]
```

## 주의사항

- AGENTS.md 스타일은 프로젝트의 기존 AGENTS.md를 따른다. 먼저 읽고 스타일을 파악한다.
- 스킬과 AGENTS.md에 같은 내용을 중복 작성하지 않는다. 스킬은 "해당 폴더의 AGENTS.md를 읽어라"고 지시하는 구조.
- 리뷰 코멘트가 프로젝트 특화 규칙이 아닌 일반적인 코딩 실수라면, 시스템 개선 대상이 아닐 수 있다. 사용자에게 확인한다.
- Git 커밋 메시지는 프로젝트의 커밋 컨벤션을 따른다.
